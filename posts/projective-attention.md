*A projective geometry framework that reveals attention as a data-dependent neural network, and give us a fresh perspective on attention.*

---

Attention is a retrieval operation. A query searches over keys, obtains a distribution over tokens, and returns a weighted average of values. This picture is useful, but it is also a little too narrow. It makes the attention head look like a soft database lookup, where the main design questions are how to normalize the scores, how to sparsify the lookup, or how to make the computation cheaper.

$$y_i = \sum_j \text{softmax}_j\!\left(\frac{q_i^\top k_j}{\sqrt{d_k}}\right) v_j$$

I'd like to introduce a new perspective to look at this operation.  Before attention retrieves anything, it constructs a geometric object. The query-key dot products do not merely compare tokens; they evaluate the position of one vector against a collection of token-defined hyperplanes. Every key is a normal vector. Every query is tested against all of those normals. The result is not just a probability distribution, but an activation pattern induced by the current context.

This suggests a different starting point:

> Attention is a token-conditioned hyperplane activation layer.

The hyperplane view starts from the computation attention already performs and asks what kind of neural layer it really is.

Standard attention is usually written as:

$$
y_i = \sum_j \alpha_{ij} v_j
$$

where:

$$
\alpha_{ij} = \operatorname{softmax}_j(q_i^\top k_j).
$$

The usual interpretation is that \(q_i^\top k_j\) is a similarity score. But if \(k_j\) is viewed as a normal vector, it defines a hyperplane in query space:

$$
H_j = \{q : q^\top k_j = 0\}.
$$

The score \(q_i^\top k_j\) is therefore also a signed incidence quantity. It tells us which side of the key-plane the query lies on, and how far it lies in that normal direction. In this view, attention does not merely retrieve from a set of tokens. It evaluates a query against an arrangement of hyperplanes generated by those tokens.

Abstractly, an attention head has the form:

$$
f(q_i) = \sum_j v_j \, \phi(q_i^\top k_j),
$$

where standard attention chooses:

$$
\phi = \operatorname{softmax}.
$$

This makes an attention head resemble a dynamic one-hidden-layer neural network:

- keys \(k_j\) are hidden-unit normals,
- queries \(q_i\) are inputs,
- values \(v_j\) are output weights,
- attention weights are activations.

The difference from an ordinary MLP is that the hidden units are not fixed parameters. They are supplied by the sequence itself. Every token contributes both a key-normal and a value-output vector. The head is therefore a context-generated activation layer: the current sequence defines the hyperplanes, the query produces an activation code over those hyperplanes, and the values decode that code into the output.

This reframing matters because it changes the design space. If attention is only retrieval, then softmax is almost forced: we want a distribution over memory locations. But if attention is a hyperplane activation layer, softmax is only one possible activation function over incidences. We can ask:

- Should the activation be dense like softmax, or sparse like ReLU?
- Should the two sides of a key-hyperplane have different meanings?
- Should key scale matter, or should only the projective direction matter?
- Should the whole attention row be treated as an activation code?
- Should the model receive explicit projection features from the query's geometry relative to the key-planes?
- Should violations of key-defined halfspaces produce vector-valued corrections?

The rest of this note explores that design space.

### The Projective Geometry Framework

The projective part of the story begins with a simple observation: a hyperplane normal is defined only up to scale. The sets

$$
H_j = \{q : q^\top k_j = 0\}
$$

and

$$
H_j = \{q : q^\top (c k_j) = 0\}
$$

are the same hyperplane for any nonzero scalar \(c\). If we take the hyperplane interpretation seriously, then the direction of \(k_j\) is more fundamental than its norm.

Vanilla dot-product attention does not respect this projective equivalence:

$$
q^\top (c k_j) = c(q^\top k_j).
$$

Scaling a key changes the logit, the softmax distribution, and therefore the output. This may be useful as a learned confidence or temperature effect, but it also entangles two roles: the direction of the hyperplane and the scale of the score.

A projective attention layer can separate these roles by choosing a representative for each direction. The simplest gauge is normalization:

$$
\hat q_i = \frac{q_i}{\|q_i\|},
\qquad
\hat k_j = \frac{k_j}{\|k_j\|}.
$$

The incidence score becomes:

$$
s_{ij} = \tau \hat q_i^\top \hat k_j.
$$

Here \(\hat k_j\) is the key-plane normal direction, \(\hat q_i\) is the query direction, and \(\tau\) controls sharpness. This is not pure projective geometry in the classical sense, because we still use a metric to normalize vectors and measure angles. But it is the practical projective geometry of neural attention: directions define hyperplanes, normalization fixes the gauge, and learned scale controls activation strength.

From this perspective, an attention head constructs an activation vector:

$$
\left[
\phi(s_{i1}),
\phi(s_{i2}),
\ldots,
\phi(s_{iT})
\right].
$$

The values then decode this activation vector:

$$
y_i = \sum_j v_j \phi(s_{ij}).
$$

The important object is no longer only the retrieved value \(y_i\). It is the activation code of the query relative to the context-defined hyperplane arrangement.

This is the main framework. Attention is not just retrieval, and it is not only a softmax over similarities. It is a learned computation over a context-generated arrangement of hyperplanes.

As I catalogue attention variants suggested by this framework, it turns out to have close or identical counterparts in the existing literature. The purpose is not to diminish these variants — the framework provides genuinely new geometric motivation for each.

The hyperplane-activation framework rewrites standard attention as

$$f(q_i) = \sum_j v_j \, \phi(q_i^\top k_j)$$

where each key $k_j$ defines a hyperplane $H_j = \{q : q^\top k_j = 0\}$, the dot product $q_i^\top k_j$ is a signed incidence score measuring which side of the hyperplane the query occupies, and $\phi$ is the activation function (softmax in the standard case). The framework decomposes attention into three independent design axes: the activation function $\phi$, the hyperplane structure (keys), and the decoding pathway (values). Each axis generates variants. Six of them have published predecessors close enough to merit detailed comparison.

Now, let's see some examples 


### Axis 1: Activation Function $\phi$

These variants change what happens to the raw incidence score $q_i^\top k_j$ before it weights the values. Standard attention uses softmax; the framework treats this as one choice among many.

- **Hard Halfspace (ReLU) Attention**

$$f(q_i) = \frac{1}{T} \sum_{j=1}^{T} v_j \cdot \text{ReLU}(q_i^\top k_j)$$

Each key hyperplane becomes a binary gate: the query is either on the positive side (contributes linearly) or on the negative side (contributes nothing). The $1/T$ normalization replaces softmax's probabilistic normalization with a simple average over active tokens.

In the hyperplane framework, this is the most literal realization of the "attention as activation" idea. The partition of query space becomes a set of convex polytopes defined by which subset of key hyperplanes the query activates — exactly the geometry of a one-hidden-layer ReLU network. Tokens on the positive side of a hyperplane contribute their value scaled by incidence magnitude; tokens on the negative side are gated off entirely. There is no competition between tokens (no softmax normalization), only independent half-space membership tests.


> Wortsman, Lee, Gilmer, and Kornblith (Google DeepMind, 2023) proposed exactly this mechanism in ["Replacing Softmax with ReLU in Vision Transformers"](https://arxiv.org/abs/2309.08586) (arXiv:2309.08586). They systematically tested ReLU, squared ReLU, GELU, softplus, and identity as point-wise softmax replacements in Vision Transformers trained on ImageNet-21k. Their central finding was that the $1/L$ sequence-length normalization is critical: without it, ReLU attention degrades significantly because the expected activation magnitude grows with sequence length; with it, ReLU attention approaches or matches softmax scaling behavior as a function of compute. They also found that an additional learned scalar per head (LayerScale-style) further stabilizes training.

> The hyperplane framework explains *why* $1/T$ works: it keeps the expected activation per hidden unit at $O(1)$ as the "network width" (sequence length) grows, the standard initialization condition for stable signal propagation in ReLU networks. This is a clean geometric motivation that the original paper did not provide — Wortsman et al. discovered the normalization empirically.

> A deeper theoretical connection comes from Lai, Lim, and Liu (2024), ["Attention is a Smoothed Cubic Spline"](https://arxiv.org/abs/2408.09624) (arXiv:2408.09624). They prove that ReLU attention computes a piecewise cubic polynomial whose breakpoints are determined by a hyperplane arrangement in query space — precisely the polytope structure the framework describes. Standard softmax attention is the $C^\infty$ smoothing of this piecewise function. This paper provides the formal mathematical grounding for the claim that "softmax is one smooth activation; ReLU is the piecewise-linear one."

**Verdict: identical mechanism.** The framework adds geometric motivation and connects to the spline theory, but the attention formula, normalization, and empirical behavior are fully characterized in Wortsman et al. 2023.

- **Adaptive Temperature Attention**

Per-head version:

$$\alpha_{ij} = \text{softmax}_j(\tau_h \cdot q_i^\top k_j)$$

Per-query version:

$$\alpha_{ij} = \text{softmax}_j(\tau(q_i) \cdot q_i^\top k_j)$$

where $\tau_h$ is a learnable per-head scalar and $\tau(q_i)$ is a small function (typically a linear projection) of the query.

In the hyperplane framework, the temperature controls the sharpness of the activation code's discrimination between hyperplane regions. Low temperature (high $\tau$) makes each hyperplane boundary a near-binary decision — the query is clearly on one side or the other. High temperature (low $\tau$) produces smooth blending across regions. The insight is that different heads may need different partition granularities: some heads perform precise retrieval (sharp boundaries), while others aggregate broad context (smooth blending). Making temperature query-dependent goes further — the same head can be sharp for some inputs and smooth for others.

> Zhang, Chang, Li, Roy-Chowdhury, Chen, and Oymak (NeurIPS 2024) proposed precisely the per-query adaptive temperature in ["Selective Self-Attention: Enhancing Transformer through Principled Context Control"](https://arxiv.org/abs/2411.12892) (SSA, arXiv:2411.12892). SSA adds a learnable inverse-temperature function $\tau(q)$ applied to each query before softmax. Their key contribution is explicitly decoupling "semantic similarity" (what the QK dot product measures) from "contextual sparsity" (how sharply the model focuses), arguing these are independent design goals that standard $1/\sqrt{d_k}$ scaling conflates. SSA adds fewer than 0.5% extra parameters via weight sharing and shows consistent improvements on language modeling benchmarks up to 1.3B parameters.

> The per-head (non-query-dependent) version has an even older precedent in Henry, Dachapally, Pawar, and Chen (Findings of EMNLP 2020), ["Query-Key Normalization for Transformers"](https://aclanthology.org/2020.findings-emnlp.379/) (QKNorm). QKNorm replaces the fixed $1/\sqrt{d_k}$ scaling with a learnable per-head scalar $\gamma$ after $\ell_2$-normalizing both queries and keys. This is mathematically equivalent to a per-head learnable temperature applied to cosine similarity. The normalization step constrains the hyperplane geometry to the unit sphere, and the learned $\gamma$ controls how sharply the model discriminates along angular distance.

The hyperplane framework's contribution here is the *geometric explanation*: temperature controls partition smoothness, and different heads learning different temperatures corresponds to the model learning a mixture of coarse-grained and fine-grained hyperplane arrangements. SSA's "decoupling similarity from sparsity" is a special case of this broader geometric claim. But the mechanism itself — learnable temperature, either per-head or per-query — is fully published.

**Verdict: identical mechanism.** SSA implements the per-query version exactly. QKNorm implements the per-head version. The hyperplane partition-smoothness interpretation is new.

- **Signed Attention**


$$\alpha_{ij} = \frac{q_i^\top k_j}{\sum_{l=1}^{T} |q_i^\top k_l|}, \quad f(q_i) = \sum_j \alpha_{ij} \, v_j$$

The denominator normalizes by total absolute incidence, preserving the sign of each score. Tokens on the positive side of a key hyperplane contribute positive weights; tokens on the negative side contribute negative weights that actively subtract their values from the output.

In the hyperplane framework, standard softmax discards half the information in the incidence score by mapping everything to $[0,1]$. The sign of $q^\top k_j$ tells the model *which side* of the hyperplane the query is on, but softmax treats a score of $-5$ and $+1$ as "low attention" and "moderate attention" rather than "strongly on the negative side" and "weakly on the positive side." Signed attention preserves this distinction. The output is no longer restricted to the convex hull of the values (the softmax simplex constraint); it can be any point in the affine span, because negative weights allow extrapolation beyond the values.

Two recent papers achieve negative attention weights through different constructions.

> Lv, Xie, Li, Liao, Sun, Kang, Wang, and Yan (2024) proposed Cog Attention in ["More Expressive Attention with Negative Weights"](https://arxiv.org/abs/2411.07176) (arXiv:2411.07176). Their formulation is $\phi(QK^\top) = \text{sign}(QK^\top) \odot \text{softmax}(|QK^\top|)$: the magnitude is normalized by softmax (ensuring the absolute values form a probability distribution), but the sign of the original dot product is preserved. They demonstrate that negative weights enable a single head to simultaneously perform "copy" and "delete" operations that would require separate heads with softmax attention, and show gains on language modeling and long-context retrieval. This is close in spirit — both preserve the sign of $q^\top k$ — but the normalization differs: Cog Attention uses softmax on absolute values, while signed attention uses L1 normalization of the raw scores.

> Ye, Dong, Xia, Sun, Zhu, Huang, and Wei (Microsoft Research, ICLR 2025 Oral) proposed the Differential Transformer in ["Differential Transformer"](https://arxiv.org/abs/2410.05258) (arXiv:2410.05258). Their mechanism is $\text{Attn} = \text{softmax}(Q_1 K_1^\top / \sqrt{d}) - \lambda \cdot \text{softmax}(Q_2 K_2^\top / \sqrt{d})$, where two separate QK projections produce two softmax maps and the difference yields effective negative weights. This achieves a similar outcome — signed attention weights — but through subtraction of two positive distributions rather than direct sign preservation. The Differential Transformer uses twice the QK parameters and requires two softmax computations; signed attention requires only one dot product and an L1 normalization.

The hyperplane framework unifies these: both Cog Attention and Diff Transformer are ways of exploiting the full signed incidence rather than just the positive half. The specific L1-normalized construction proposed here is a simpler mechanism that achieves the same conceptual goal but is not identical to either published method. The normalization choice (L1 vs. softmax-of-absolutes vs. difference-of-softmaxes) affects the weight distribution and training dynamics in ways that require empirical comparison.

**Verdict: close but distinct.** The idea of negative attention weights is published in multiple forms. The specific L1-normalized signed formulation is a distinct construction that has not been tested. The difference is in the normalization, not the conceptual insight.

### Axis 2: Hyperplane Structure (Keys)

These variants change the hyperplane arrangement itself — how the keys are constructed, structured, or accessed.

- **Polytope Attention (Hashed Hyperplane Regions)**

$$h(q_i) = \text{sign}\!\left([q_i^\top k_1, \; q_i^\top k_2, \; \ldots, \; q_i^\top k_T]\right)$$

Queries with the same binary sign pattern $h(q_i) \in \{-1, +1\}^T$ occupy the same polytope in the hyperplane arrangement. Attention is computed only within groups of tokens sharing the same hash bucket, reducing complexity from $O(T^2)$ to $O(T \log T)$.

In the framework's language, standard attention computes the full activation code (a continuous vector of incidence scores) and then decodes it through softmax + values. Polytope attention binarizes the activation code to a sign pattern and groups queries by code identity. Two queries in the same polytope — on the same side of every hyperplane — are treated as having the same activation and should produce similar outputs. This is a discrete approximation of the continuous activation code.


> Kitaev, Kaiser, and Levskaya (ICLR 2020) proposed exactly this idea in ["Reformer: The Efficient Transformer"](https://arxiv.org/abs/2001.04451) (arXiv:2001.04451). Reformer uses locality-sensitive hashing (LSH) to bucket query-key vectors (with shared Q=K projections) and restricts attention to within-bucket pairs. The specific LSH scheme is based on random hyperplane projections (Charikar-style): project the query onto $r$ random directions, take the sign of each projection, and use the resulting binary string as the hash code.

> The hyperplane framework reveals that Reformer's LSH is literally a random sampling of hyperplanes from the key arrangement plus sign-pattern bucketing. The hash function $h(q) = \text{sign}(Rq)$ where $R$ is a random projection matrix defines a random hyperplane arrangement, and tokens are grouped by which polytope they fall in. The only structural difference is that Reformer uses *random* hyperplanes (for computational efficiency), while the framework's formulation uses the *data-dependent* key hyperplanes themselves. Using the actual key hyperplanes would give a more semantically meaningful partition but would be harder to compute efficiently (you'd need to hash with respect to a changing set of keys at each layer).

> Performer (Choromanski et al., ICLR 2021, ["Rethinking Attention with Performers"](https://arxiv.org/abs/2009.14794)) takes a related but different approach: instead of bucketing by sign pattern, it approximates the softmax kernel via positive orthogonal random features (FAVOR+), enabling linear-time attention. This is a kernel approximation rather than a polytope partition, so the connection to the hyperplane framework is looser — Performer approximates the activation function $\phi$ rather than discretizing the hyperplane arrangement.

**Verdict: identical mechanism.** The framework provides the geometric explanation (polytope membership = hash code), but the algorithm is Reformer's LSH attention. The data-dependent-key variant is a conceptual extension not explored in Reformer, but it is an implementation detail, not a new mechanism.

- **Exclusive Self Attention**

$$z_i = y_i - \frac{y_i^\top v_i}{\|v_i\|^2}\, v_i, \qquad \text{where} \quad y_i = \sum_j \alpha_{ij}\, v_j$$

Standard attention lets a token attend to itself: the diagonal entry $\alpha_{ii}$ pulls the output $y_i$ toward the token's own value $v_i$. Empirically, $y_i$ and $v_i$ end up with high cosine similarity — the output is partly a repetition of what the token already encodes. Exclusive Self Attention (XSA) removes this self-aligned component by projecting $y_i$ onto the subspace orthogonal to $v_i$, i.e., onto the hyperplane $\{y : y^\top v_i = 0\}$.

From the projective attention framework, this is a gauge-fixing operation on the *output* rather than the keys. Just as normalizing $k_j$ removes an irrelevant degree of freedom in the key space, projecting out $v_i$ removes an irrelevant degree of freedom in the output space — the direction the token could have read from itself without attending anywhere. What remains is the strictly cross-token information: the component of the aggregated context that is orthogonal to the token's own value direction. The operation costs a single inner product and a rank-one update per token, making it nearly free at inference time.

The formula has a clean projective reading. Define the projection operator onto the orthogonal complement of $v_i$ as

$$P_{v_i}^\perp = I - \frac{v_i v_i^\top}{\|v_i\|^2}.$$

Then $z_i = P_{v_i}^\perp\, y_i$. This is a data-dependent projection: the hyperplane being projected onto is defined by the token's own value vector, and that vector changes with every input. XSA is therefore a decoding-axis complement to projective attention's key-axis gauge fixing — both eliminate a direction that the framework identifies as geometrically redundant.

> Zhai (Apple ML Research, 2026) proposed this mechanism in ["Exclusive Self Attention"](https://arxiv.org/abs/2603.09078) (arXiv:2603.09078). The motivation in the paper is empirical: analysis of trained language models shows that the attention output $y_i$ and the self-value $v_i$ tend to have high cosine similarity across layers and model sizes, meaning the attention head is partially "wasting" its output capacity by re-encoding what the token already knows. Removing this component — without any additional learned parameters — consistently improves validation loss on FineWeb-100B across model sizes up to 2.7B parameters, with gains that grow with sequence length.

**Verdict: identical mechanism.** XSA implements exactly the formula above. The hyperplane framework reframes the operation as an output-space gauge fixing that is the natural dual of key normalization in projective attention — a conceptual sharpening, not a new computation.

- **Normalization in Attention**

$$\hat{q}_i = \frac{q_i}{\|q_i\|}, \quad \hat{k}_j = \frac{k_j}{\|k_j\|}, \quad \alpha_{ij} = \text{softmax}_j(\tau \cdot \hat{q}_i^\top \hat{k}_j)$$

A key hyperplane is defined only by its normal direction, not its scale: $\{q : q^\top k_j = 0\}$ and $\{q : q^\top (ck_j) = 0\}$ are the same hyperplane for any $c \neq 0$. Yet in raw dot-product attention $q^\top(ck_j) = c(q^\top k_j)$, so scaling a key shifts the logits and the softmax distribution — even though the underlying hyperplane hasn't changed. Projective attention fixes this by $\ell_2$-normalizing both queries and keys, choosing a canonical representative for each hyperplane normal. The incidence score becomes a pure angular measurement, and a learned scalar $\tau$ controls sharpness independently.

> Henry, Dachapally, Pawar, and Chen (Findings of EMNLP 2020) proposed essentially the same mechanism in ["Query-Key Normalization for Transformers" (QKNorm)](https://aclanthology.org/2020.findings-emnlp.379/). Their formulation: $\ell_2$-normalize both queries and keys along the head dimension, then scale by a learnable per-head scalar $\gamma$ instead of the fixed $1/\sqrt{d_k}$. Mathematically, this is $\text{softmax}(\gamma \cdot \hat{q}^\top \hat{k})$, which is identical to projective attention with $\tau = \gamma$. They report improved training stability and modest perplexity gains, particularly at larger depths where unnormalized attention scores can grow large.

The hyperplane framework's contribution is the *projective geometry* language: the reason normalization helps is gauge fixing, not just training stability. This is a conceptual sharpening of an existing observation, not a new mechanism.

**Verdict: identical mechanism.** QKNorm is the same formula. The "projective gauge" interpretation is novel framing.

### Cross-Cutting: Reframing the Standard Mechanism

- **Activation-Code Attention**

$$a_i = \phi(q_i K^\top) \in \mathbb{R}^T, \quad y_i = a_i V$$

The activation code $a_i$ is the vector of activation values (attention weights) for query $i$ across all keys. Standard attention computes exactly this; the reframing emphasizes that $a_i$ is the learned representation and $V$ decodes it, rather than thinking of attention as "soft retrieval from a value store."

In the hyperplane framework, $a_i$ encodes the query's position relative to all $T$ key hyperplanes. It is a $T$-dimensional code where each coordinate measures the incidence of the query with one hyperplane. The values $V$ are a learned linear decoder that maps this code to the output. This makes attention look like a one-hidden-layer neural network: keys are hidden-unit weight vectors, the activation code is the hidden representation, and values are the output weights. The difference from a standard MLP is that the hidden units (keys) and output weights (values) are data-dependent — they change with every input sequence.

This reframing has no computational implications (it changes no operations), but it has significant conceptual ones: it suggests that the activation code itself is an object worth analyzing, that different $\phi$ choices define different activation code geometries, and that the value projection is a decoder that could be designed independently of the activation.


> Schlag, Irie, and Schmidhuber (ICML 2021) made essentially the same observation in ["Linear Transformers Are Secretly Fast Weight Programmers"](https://arxiv.org/abs/2102.11174) (arXiv:2102.11174). They proved that linearized attention (with $\phi$ factored as a kernel) is equivalent to a fast-weight network: the model stores an outer-product memory $W^{(t)} = \sum_i v_i k_i^\top$ such that the output is $y = W^{(t)} q$. This is exactly "keys define the network, queries are inputs, values are output weights." Their analysis focuses on the linear case (where the kernel decomposition enables the fast-weight reformulation), but the conceptual point — attention implements a data-dependent one-hidden-layer network — is the same.

> Tsai, Bai, Yamada, Morency, and Salakhutdinov (EMNLP-IJCNLP 2019) arrived at a related view in ["Transformer Dissection: An Unified Understanding for Transformer's Attention via the Lens of Kernel"](https://aclanthology.org/D19-1443/). They recast attention as Nadaraya-Watson kernel regression and state explicitly that "the softmax function is by no means essential or necessary — it is merely one choice of kernel." The kernel view treats $\phi$ as a similarity kernel and $V$ as the function values at the data points; the output is a kernel-weighted average. This is mathematically equivalent to the activation-code view but uses different vocabulary (kernel smoothing vs. neural network activation).

The hyperplane framework's specific contribution is the geometric language — "activation code" relative to a "hyperplane arrangement" — which connects attention to the rich theory of hyperplane arrangements, piecewise-linear functions, and ReLU network expressivity. This connection was made rigorous by Lai, Lim, and Liu (2024) in the [cubic spline paper](https://arxiv.org/abs/2408.09624). But as a computational mechanism, activation-code attention is standard attention with a new name.

**Verdict: identical mechanism (reframing only).** The activation-code view is implicit in Schlag et al. 2021 and Tsai et al. 2019. No implementation change is needed; the value is conceptual.

The pattern is consistent: the hyperplane framework generates these variants naturally from geometric first principles, but the ML community has already discovered each mechanism through empirical search, efficiency engineering, or kernel theory. The framework's value for these six variants is not novelty of mechanism but novelty of explanation — it provides a unified geometric lens that connects mechanisms previously understood in isolation (ReLU attention, LSH attention, cosine attention, adaptive temperature) as different choices along the same design axis.

The framework's genuine experimental novelty lies elsewhere: in the decoding axis (HPA, dual-code, projection features) and in the cross-cutting variants (margin-gated, two-sided hyperplane, conjugate pairs), where published precedents are either absent or structurally different. Those variants should receive the bulk of experimental compute.

<!-- ## Why This Framework Matters

The hyperplane activation framework isn't just a reinterpretation — it generates concrete predictions and design insights.

### Attention patterns are hyperplane arrangements, not similarity matrices

The standard narrative says attention computes "similarity" between queries and keys. The hyperplane view says attention computes the query's position relative to a set of data-dependent decision boundaries. These are different mental models with different implications.

Similarity is symmetric and monotonic: more similar → higher weight. Hyperplane geometry is directional and structural: the weight depends on *which side* the query is on, and the arrangement of *all* hyperplanes simultaneously determines the activation pattern through softmax's normalization. A query can have high dot product with $k_j$ and still give low attention weight to token $j$ if other keys produce even higher dot products — not because the query is "less similar" to token $j$, but because it has crossed deeper into other tokens' half-spaces.

### The capacity of attention scales with sequence length

In a ReLU network with $n$ hidden units in $d$ dimensions, the number of linear regions is bounded by $O(n^d)$. By analogy, an attention head with $T$ tokens in $d_k$ dimensions can express up to $O(T^{d_k})$ distinct activation patterns. As sequence length grows, the number of hyperplanes grows, and the partition of query space becomes finer.

This gives a geometric explanation for why attention scales well with context length: longer sequences provide more hyperplanes, enabling finer partitions and more precise activation codes. It also suggests why attention becomes expensive — the network is literally growing its hidden layer with every additional token.

### Softmax temperature controls partition smoothness

The scaling factor $1/\sqrt{d_k}$ in attention controls the "temperature" of the activation function. Low temperature (small scale) makes softmax sharper, pushing toward hard halfspace assignments — the activation code becomes more like a binary sign pattern. High temperature makes softmax flatter, blending activations across many hyperplanes.

In ReLU networks, the sharpness of the activation is fixed (it's a hard threshold). In attention, it's a design choice. This connects to findings on attention entropy: well-trained models develop heads with both sharp and diffuse attention patterns — corresponding to heads that need precise hyperplane discrimination and heads that need smooth blending. -->



### Looking Forward: What the Framework Suggests

The hyperplane activation view opens several directions.

**Attention as function approximation.** If an attention head is a one-hidden-layer network with $T$ data-dependent units, classical approximation theory applies. Universal approximation results for single-hidden-layer networks tell us that with enough hidden units (tokens), an attention head can approximate any continuous function of the query. The approximation quality depends on how well the key hyperplanes tile query space — which the model controls by learning the key projection $W_K$.

**Connections to kernel methods.** The activation code $\alpha_i = \text{softmax}(q_i^\top K / \sqrt{d_k})$ defines an implicit kernel between queries. Two queries with similar activation codes — meaning they occupy the same region in the hyperplane arrangement — will produce similar outputs. This connects attention to kernel methods, where the kernel measures similarity in a feature space defined by the hyperplane arrangement rather than in the original embedding space.

**Hyperplane-aware modifications.** Once we recognize keys as hyperplane normals, we can ask: what if the output retrieves information that the hyperplane activation *didn't* capture — the component of the value orthogonal to the key direction? This is exactly the intuition behind Hyperplane-Projection Attention, where the correction strips key-aligned components before aggregation, forcing the output to carry only the information that the activation code couldn't already encode. The hyperplane framework provides a natural language for designing such modifications: they are operations on the relationship between the query's position in the hyperplane arrangement and the information retrieved from each hyperplane.

**Interpreting trained attention.** The capacity of attention scales with sequence length. In a ReLU network with $n$ hidden units in $d$ dimensions, the number of linear regions is bounded by $O(n^d)$. By analogy, an attention head with $T$ tokens in $d_k$ dimensions can express up to $O(T^{d_k})$ distinct activation patterns. As sequence length grows, the number of hyperplanes grows, and the partition of query space becomes finer. This gives a geometric explanation for why attention scales well with context length: longer sequences provide more hyperplanes, enabling finer partitions and more precise activation codes. It also suggests why attention becomes expensive — the network is literally growing its hidden layer with every additional token.

### Finally

Attention is not merely a soft lookup table. It is a token-conditioned hyperplane activation layer: a one-hidden-layer neural network whose hidden units, weight vectors, and output weights are all instantiated from the input data.

Each key defines a hyperplane in query space. The dot product between query and key is a signed incidence score — which side of the hyperplane, and how far. Softmax converts the vector of incidence scores into an activation code: a probability distribution over tokens that encodes the query's position in the hyperplane arrangement. Values decode this activation code into the output.

The framework is not merely a metaphor. It is the literal geometry of what $q^\top k$ computes: the position of a point relative to a hyperplane. Everything else — similarity scores, attention patterns, soft retrieval — follows from that single geometric fact.
